nanoc...then host it on Heroku

It was simple enought, I wanted to know what this nanoc thing was about so I went ahead and installed the gem and got to typing. After some time I realized that the site would need to live somewhere other than my laptop so I explore possible hosting options...from github pages to setting up my own little server from a cloud service. But what I really wanted was the extremely simple code and push functionality that Heroku Toolbelt provided. After running out of patience with Dokku, (still on the back-burner), I went with Heroku as a temporary solution.

At this point, I've decided to take this a step further and host my little nanoc site on Heroku. Beacuase simply uploading the files made too much sense (and I could not figure out how to do it), I set it up as a Sinatra static server. So not the most efficient way to have a static application up and running but I'm all smiles for it. :).
